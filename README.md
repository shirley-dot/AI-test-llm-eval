本项目是一个围绕大语言模型（LLM）的自动化评估与测试工具集，旨在系统性地验证模型在安全、任务表现和内容质量三个维度的能力。

项目通过集成 Prompt 对比评测、红队测试（Red Teaming）、批量用例生成和幻觉检测（Faithfulness），构建了一个从“模型选型”到“内容交付”的完整质量保障流水线。

一、Prompt 评测（promptfoo eval）
为了找出针对“需求 → 测试用例”转化效果最佳的 Prompt，本模块设计了三种不同方向的 Prompt 模板，并设置 llm-rubric（百分制） 进行自动化评分。

评分围绕以下四个维度展开：

正向覆盖度：是否覆盖了需求中所有“正常流程”和“主功能点”？

负向覆盖度：是否覆盖了“边界值”（如 0、空值、最大值）、“异常输入”（如特殊字符、超长文本）和“并发/重复操作”？

数据与规则完整性：针对密码生成、金额计算、优惠券抵扣等涉及数据的场景，是否验证了数据精度和业务规则？

结果可验证性：测试步骤是否清晰？是否给出了明确的、可量化的预期结果（而不是模糊的描述）？

通过该评测，您可以从多个候选 Prompt 中筛选出综合得分最高的“最佳 Prompt”，为后续批量生成奠定基础。

二、红队测试（promptfoo redteam run）
红队测试用于主动探测模型的潜在安全漏洞和对抗性风险。本项目的红队配置支持同时对多个模型进行攻击测试，便于横向对比。

应用场景：如果您在业务中同时使用了多个模型（如通义千问、豆包等），建议用相同的红队配置对所有模型进行测试，以横向对比其安全表现，辅助最终的模型选型决策。

红队测试与 Prompt 评测相互补充——前者关注安全性，后者关注任务效果，共同构成模型能力的全面评估。

三、批量生成测试用例（generate_cases.py）
通过评测获得“最佳 Prompt”后，使用批量生成脚本 generate_cases.py，可自动为大量需求生成对应的测试用例。

脚本支持以下三种输出格式，便于在不同场景下使用：

generated_cases.json：结构化数据，便于程序处理和二次开发。

generated_cases.md：Markdown 格式，适合直接进行人工评审和文档分享。

generated_cases.csv：表格格式，方便导入 Excel 或测试管理平台（如 Jira、禅道）。

四、幻觉检测（hallucination_check.py）
为确保生成的测试用例内容可靠，本项目集成了幻觉检测（Faithfulness） 机制，通过独立脚本 hallucination_check.py 执行。

检测目标：确保测试用例的每一项内容都与原始需求完全一致，不引入任何事实性矛盾或凭空捏造的信息。

检测方式：通过 FaithfulnessMetric 将生成的用例与原始需求进行事实比对，输出忠实度评分。
