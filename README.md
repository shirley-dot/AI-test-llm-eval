# AI-test-llm-eval
 一、红队测试：如果你在业务中同时使用了多个模型（如通义千问 + 豆包），建议用相同的红队配置测试所有模型，横向对比安全表现，辅助选型决策。
 二、 Prompt 评测：使用三种方向的Prompt，针对相同的需求生成测试用例，设置llm-rubric（百分制），通用版本针对以下四个维度进行评分，从而获得最佳Prompt
        1. 正向覆盖度：是否覆盖了需求中所有“正常流程”和“主功能点”？
        2. 负向覆盖度：是否覆盖了“边界值（如0、空值、最大值）”、“异常输入（如特殊字符、超长文本）”和“并发/重复操作”？
        3. 数据与规则完整性：针对密码生成、金额计算、优惠券抵扣等涉及数据的场景，是否验证了数据精度和业务规则？
        4. 结果可验证性：测试步骤是否清晰？是否给出了明确的、可量化的预期结果（而不是模糊的描述）？
 三、批量生成测试用例：使用最佳Prompt，使用脚本批量生成测试用例，并且以三种格式输出:
    ├── generated_cases.json
    ├── generated_cases.md
    └── generated_cases.csv
 四、幻觉检测：确保测试用例的内容都与原始需求完全一致，无事实性矛盾
